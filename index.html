2. Hotel onboarding

Onboarding starts with an API call to dextr-marketing-svc. The service creates the hotel row, calls deepagent to scrape the website, and saves profile + categorized images. No knowledge-base file in Phase 1 — scraped facts go into profile_data jsonb.

Onboarding API (entry point)

MethodPathPurpose
POST/api/v1/hotelsCreate hotel + start scrape job
GET/api/v1/hotels/:idPoll onboard_status
 
POST /api/v1/hotels
{
  "name": "Grand Plaza Hotel",
  "website_url": "https://grandplaza.com",
  "timezone": "America/New_York",
  "alfred_hotel_id": "alf_123",
  "daisy_hotel_id": "dsy_456",
  "manny_hotel_id": null
}

Response: { "hotel_id": "...", "onboard_status": "scraping" }

At least one of alfred_hotel_id, daisy_hotel_id, or manny_hotel_id should be set so sign-in-with-services can resolve the hotel later.

URL scrape flow

Entry: POST /api/v1/hotels — marketing-svc receives hotel details, creates hotels row, onboard_status = scraping
marketing-svc — enqueues jobs type=scrape, calls deepagent
deepagent — crawls website_url, extracts profile facts, brand tone, images
deepagent — uploads images to S3, assigns category per image; calls PUT profile + POST assets
marketing-svc — rebuilds assets_by_category cache → onboard_status = ready

3. User flow journey

From login to sent campaign — what the hotel user experiences and what happens behind the scenes.

3.1 Login to dashboard

The user signs in through the Dextr core auth service. The login API is POST https://core.dextr.ai/api/auth/sign-in-with-services. A successful response returns a JWT and a list of available products for that hotel (e.g. Alfred, Daisy, Email Marketing). If Email Marketing appears in that list, the hotel is entitled to use this product.

When the user opens the Email Marketing Agent, the dashboard sends the JWT to marketing-svc on every request. The token carries the hotel's product ids (alfred_hotel_id, daisy_hotel_id, and/or manny_hotel_id). marketing-svc validates the token with core auth, then looks up the hotels table — a row must match at least one of those ids. If a match is found and onboard_status = ready, the dashboard loads.

User signs in → POST https://core.dextr.ai/api/auth/sign-in-with-services
Response: JWT + products[] — must include Email Marketing
User opens Email Marketing Agent → GET /api/v1/me with Authorization: Bearer <jwt>
marketing-svc validates JWT, matches alfred_hotel_id / daisy_hotel_id / manny_hotel_id on hotels
Match found + onboard_status = ready → dashboard renders. No local users table.
Prerequisite: the hotel must be onboarded via POST /api/v1/hotels first, with the correct product hotel id(s) stored. A valid JWT with no matching hotels row → 403.

3.2 Chat — describe the campaign

User interaction starts with a message in the chat text box on the Marketing Agent home screen. The user sees "Create a new campaign…" and types their campaign idea — for example, "Generate email marketing for Mother's Day, referring to the spa offers" — then clicks send.

Recommendation cards on the dashboard are Phase 2 — not shown in Phase 1.

StepWhat happens
1POST /api/v1/campaigns — draft row with user_prompt
2POST /api/v1/campaigns/:id/messages — chat message stored
3UI shows the layout picker on the left — deepagent is not called yet

3.3 Select a template

The left panel shows layout tiles: Showcase, Announcement, Highlights, Promotion, Story, and Auto. The right panel shows "No campaign yet" until a layout is chosen and generated.

When the user clicks a tile (e.g. Showcase), the UI handles it directly — it does not send a chat message to the agent. The UI highlights the tile, sets template_id on the campaign, and triggers generation.

StepWhat happens
1PATCH /api/v1/campaigns/:id — sets template_id + template_key
2UI highlights selected tile; optional system row in campaign_messages
3POST /api/v1/campaigns/:id/generate — deepagent fills template JSON
4Renderer merges template HTML + campaign_json → html_url
5Right panel shows preview v1

deepagent receives: profile_data + assets_by_category + user prompt + template json_schema + prompt.

3.4 Preview & chat edits

Once generated, the email preview appears on the right with subject line, desktop/mobile toggle, and theme options. The user can keep refining via the chat box on the left ("Describe your campaign…").

User actionBackend
"Update the title to Summer Spa Special"POST /messages → deepagent patches JSON/HTML → new html_url → preview refreshes
"Increase heading font by 2px"deepagent edits the latest HTML document at html_url
Click SavePUT /api/v1/campaigns/:id — persists html + json, status = draft

Mixing layout from one template with fonts from another (e.g. Showcase + Highlights font) is Phase 2.

3.5 Recipients, schedule & send

After the user is happy with the preview, they upload a recipient CSV (or add emails manually), then schedule or send immediately.

User actionAPIEffect
Upload CSVPOST /api/v1/campaigns/:id/recipientsRows in campaign_recipients
SchedulePOST /api/v1/campaigns/:id/schedulestatus = scheduled, jobs row for cron
Send nowPOST /api/v1/campaigns/:id/sendstatus = sending, background send job starts

Send job: worker loops campaign_recipients where send_status = pending, sends each email via SES with tracking pixel and link redirects, updates jobs.progress_percent, and sets campaigns.status = sent when all recipients are processed.

3.6 Status & analytics

User can ask in chat "What's the status of my campaign?" — marketing-svc reads campaigns.status, job progress, and recipient counts, then returns an assistant message. No deepagent call needed.

Campaign list shows open/click rates from tracking_events via GET /api/v1/campaigns/:id/stats.

Database — 10 tables

Phase 1 schema. No users table (central auth). No knowledge_bases table (profile_data jsonb instead).

  1. hotels — tenant, linked by alfred_hotel_id / daisy_hotel_id / manny_hotel_id
  2. hotel_profiles — profile_data + assets_by_category + sender
  3. assets — one image per row, category column
  4. email_templates
  5. campaigns
  6. campaign_messages — chat thread
  7. campaign_recipients
  8. campaign_links
  9. tracking_events
  10. jobs — scrape + send workers
